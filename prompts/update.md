# 문서 병합/업데이트 지침

기존 문서를 훼손하지 않고 최신 정보를 병합하는 지침.

## 분기 처리
| 상황 | 처리 |
|------|------|
| 두 파일 모두 존재 | 아래 병합 플로우 |
| 두 파일 모두 없음 | add-knowledge 플로우로 전환 |
| 한 파일만 존재 | 있는 건 update, 없는 건 add |

## 병합 순서
1. 기존 파일 읽기 + 새 정보(리서치 결과, 링크, 텍스트) 비교
2. 대화창에 브리핑:
   - **공통점:** 유지되는 핵심 개념
   - **차이점:** 새 접근 방식, 충돌 내용
   - **변경 요약:** 어디가 어떻게 바뀌는지
3. 아래 병합 정책 적용하여 파일 수정
4. Frontmatter `date_updated` 갱신, `sources` 추가

## 병합 정책 (충돌 시 우선순위)
1. **수식/정의:** 기존 보존 + `> 🔁 업데이트 주석`으로 새 정보 병기
2. **실험 수치:** 최신 출처 우선. 기존 수치는 ~~취소선~~ + 출처 보존
3. **아키텍처/구현:** `v1 방식` / `v2 방식 (최신)` 병기. 삭제 금지
4. **출처 없는 새 정보:** 본문 반영 보류, Changelog에 `[출처 미확인]`만 기록

## Changelog
파일 최하단에 추가 (이미 있으면 최상단 항목으로):
```
## 🕒 Changelog
- **[YYYY-MM-DD]:** {추가/수정 내용 1줄} | 출처: {URL/논문명}
```
충돌 처리 시 `⚠️ 충돌 처리:` 항목으로 적용 원칙 명시.
